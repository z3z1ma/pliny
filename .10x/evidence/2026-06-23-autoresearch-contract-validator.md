# Autoresearch Contract Validator Validation

## What Was Observed

The contract validator implementation produced:

- `autoresearch/validate.py`
- `autoresearch/tests/test_validate.py`
- README validation-command documentation in `autoresearch/README.md`

The checked-in static contracts validated successfully:

```text
$ python3 autoresearch/validate.py
autoresearch contracts valid
```

The focused unittest suite passed:

```text
$ python3 -m unittest discover -s autoresearch/tests
...
----------------------------------------------------------------------
```

The tests include invalid copied-contract cases that remove `S009` and `SCN-015`
and assert that `validate.validate_contracts(...)` reports the missing ID errors.

produced no output:

## Limits

This evidence does not show that:

- A real score artifact validates against the schema.
- The validator is a complete JSON Schema implementation.
- Offline scoring exists.
- Subject-agent experiments, transcript scoring, reports, or harness integrations
  exist.
- Future static contract changes will remain valid without rerunning the
  validator.

Generated `__pycache__` files were created by Python during tests and are ignored
by git; they were excluded from the source ASCII scan.
