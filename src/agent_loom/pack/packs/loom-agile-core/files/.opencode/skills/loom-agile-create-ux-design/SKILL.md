## Outcomes
- Key user flows
- Information architecture
- UI direction (components, states, accessibility)

## Procedure
1) Clarify target users and success metrics
2) Define flows (happy path + critical edge cases)
3) Define IA and navigation
4) Define UI direction
   - states
   - error handling
   - responsive behavior
   - accessibility constraints
5) Persist and link from the ticket
