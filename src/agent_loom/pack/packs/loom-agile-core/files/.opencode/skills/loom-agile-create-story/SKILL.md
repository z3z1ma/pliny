## Story format
- Title
- Context
- Acceptance criteria
- Task breakdown
- Test/verification notes

## Procedure
1) Confirm goal and constraints
2) Write acceptance criteria first
3) Break down tasks in the smallest verifiable steps
4) Identify dependencies
5) Persist into a ticket note or a repo doc, then link from the ticket
