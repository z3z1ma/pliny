## Outcomes
- A project overview doc
- A “how to run” doc (tests/build/dev)
- Pointers to key modules/components

## Procedure
1) Discover how the repo is structured
2) Identify the key entrypoints
3) Document:
   - what the project does
   - how to run it
   - how to test it
   - where the important code is
4) Validate docs against reality (commands actually work)
5) Add a ticket note linking the docs
