## When to use
- The work is stuck.
- Requirements changed.
- Implementation revealed new constraints.

## Procedure
1) Restate the goal and current state
2) Identify drift
   - scope creep
   - misunderstood requirements
   - technical constraint
3) Propose a correction
   - reduce scope
   - split into tickets
   - revise PRD/architecture
4) Update Loom
   - ticket note with the corrected plan
   - create/link new tickets if needed
