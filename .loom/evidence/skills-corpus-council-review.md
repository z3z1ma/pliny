# Artifacts

- Council task id: `ses_21866e25cffePOACghVwgMjBTa`.
- Review target: `skills/`.
- Context target: `README.md`.
- The council's top-level conclusion: the corpus is unusually coherent and should
  be sharpened through consistency, grammar, routing, and validation improvements
  rather than through a runtime, daemon, database, CLI, or new ontology.
- High-priority improvement findings:
  - Surface `loom-drive` in README, protocol examples, and bootstrap routing.
  - Reconcile README's linear outer-loop diagram with the bootstrap backbone.
  - Make cold-start and post-compaction resume first-class workspace guidance.
  - Formalize `OBJ-*` objective coverage grammar.
  - Normalize packet and handoff grammar across Ralph, critique, wiki, and drive
    handoffs.
  - Decide how strict ticket `change_class` and `risk_class` should be.
  - Add a canonical `kind:` / ID / path table.
  - Reduce duplicated atlas, retrospective, spike, and sketch doctrine.
  - Explicitly ban generic scratchpads as junk-drawer substitutes.
  - Tighten skill metadata conventions.
- Additional underdeveloped areas identified by the council:
  - pre-compaction checkpoint discipline,
  - external-reference lifecycle,
  - concurrent agent edit safety,
  - rollback or rejected Ralph iteration handling,
  - memory pruning cadence,
  - critique finding to follow-up ticket conversion,
  - non-Ralph packet posture.
- Risk assessment from the council:
  - High-risk: cold-start resume doctrine, `OBJ-*` grammar, packet contract
    changes, ticket required-field changes, bootstrap routing changes, and major
    `loom-drive` restructuring.
  - Medium-risk: README outer-loop rewrite, packet field renames, retrospective
    reference moves, and follow-up conversion grammar.
  - Low-risk: adding `loom-drive` to README, wording cleanup, metadata
    standardization, validation query additions, and duplicate prose
    consolidation.
