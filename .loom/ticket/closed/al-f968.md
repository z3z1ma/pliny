# Decompose team core into domain modules

## Objective alignment
`src/agent_loom/team/core.py` is the largest architecture hotspot (~6400 LOC). Splitting by domain reduces cognitive load and lowers regression risk during public launch hardening.

## Scope
- Decompose `src/agent_loom/team/core.py` into cohesive domain modules under `src/agent_loom/team/` (or `src/agent_loom/team/core/` package).
- Candidate domains:
  - run lifecycle (start/pause/resume/disband)
  - worker lifecycle (spawn/resume/retire/bounce)
  - sprint/objective management
  - inbox/send/wait flows
  - merge queue + shipping
- Keep external CLI behavior and public command signatures stable.

## Non-goals
- No new team commands.
- No tmux behavior redesign.
- No protocol changes for manager/worker messaging.

## Implementation plan
1. Define explicit module boundaries and move helpers/functions domain-by-domain.
2. Maintain a thin compatibility layer in `team/core.py` (if needed) to preserve imports while migrating call sites.
3. Update internal imports to avoid cycles; prefer explicit dependency direction (shared utilities -> domain modules).
4. Remove moved dead code from original file after each domain extraction.
5. Run team-focused regression tests, then full gates.

## Verification
- `uv run ruff check .`
- `uv run basedpyright`
- `uv run pytest tests/test_team_cli_ux.py tests/test_team_prompts.py tests/test_team_wait_checkin.py tests/test_team_spawn_headcount.py tests/test_team_ship_push.py tests/test_team_ship_ticket_sync.py`
- `uv run pytest`

## Acceptance criteria
- `team/core.py` is materially reduced and no longer concentrates unrelated domains in one file.
- New domain modules have clear responsibility boundaries and no import cycles.
- Existing team CLI behavior remains unchanged (tests pass).
- No dead wrappers/imports remain after migration.

## Risks / edge cases
- **Risk:** hidden coupling between distant helper functions in current monolith.
  - **Detection:** import cycle errors, runtime attribute errors, failing team integration tests.
  - **Mitigation:** move functionality incrementally by domain and run tests after each extraction slice.
- **Risk:** behavioral regressions in role/permission checks.
  - **Detection:** failing team CLI tests around worker/manager constraints.
  - **Mitigation:** preserve permission helper location/API and add focused regression assertions if gaps appear.
