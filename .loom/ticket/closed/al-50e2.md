# Support glob patterns in recall --scope file:

Support globbing in recall --scope when using file:/folder: kinds.

User need: allow `loom memory recall --scope file:src/agent_loom/memory/*.py` to work as a glob filter.

Behavior:
- If a `--scope` value uses kind=file or kind=folder and the value contains glob metacharacters (`*`, `?`, `[ ... ]`), treat it as a glob scope internally.
- Validation should not treat that as a concrete file existence check.
- Scope matching should allow context glob patterns to match note file/folder scopes.

Acceptance:
- `loom memory recall --scope file:src/**/*.py` filters notes by matching note file/folder scopes against the glob
- Exact `file:` and `folder:` scope matching remains unchanged
- No NOT_FOUND error for wildcard file scopes
- Tests cover recall with `--scope file:<glob>` across multiple notes
