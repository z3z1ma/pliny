# Team core: isolate wait/check-in and pane-capture loop

Objective alignment:
The wait/check-in control loop and pane capture persistence are operational concerns with time-based behavior. Keeping this logic isolated improves readability, testability, and failure analysis without changing user-facing workflows.

Scope:
- Extract wait-loop helpers and check-in gating logic from `src/agent_loom/team/core.py` into a focused operational module (for example `src/agent_loom/team/waiting.py`).
- Extract pane-capture persistence helper(s) used by auto-capture paths into the same module or a nearby operational module, keeping side-effect boundaries explicit.
- Keep `wait` command semantics, wake reasons, and manager check-in behavior unchanged.
- Expand tests in `tests/test_team_wait_checkin.py` and capture-related coverage if touched.

Non-goals:
- No changes to inbox message schema.
- No changes to tmux event primitives.
- No objective/sprint behavior changes.

Implementation plan:
1) Define module contracts for duration parsing/wake evaluation/check-in decision points.
2) Move helper logic first, then delegate from `core.wait` and related internals.
3) Preserve constants and cooldown semantics; avoid tuning thresholds in this refactor.
4) Add tests for timeout wake, inbox wake, and check-in throttling behavior.
5) Remove dead helper code/imports from core.

Verification:
- uv run pytest tests/test_team_wait_checkin.py tests/test_team_helpers.py tests/test_team_cli_ux.py
- uv run ruff check .
- uv run basedpyright

Risks/edge cases:
- Timing-sensitive tests can become flaky after extraction; use deterministic clock/control points where available.
- Wake reason normalization regressions can suppress manager check-ins unexpectedly.
- Capture persistence path errors can silently drop diagnostics if exception handling changes.

## Acceptance Criteria

- Wait/check-in and capture loop helpers are extracted into an operational module with `team/core.py` delegation.
- Wait command behavior remains unchanged for timeout/inbox wake scenarios.
- Wait/check-in regression tests pass without introducing flakiness.
