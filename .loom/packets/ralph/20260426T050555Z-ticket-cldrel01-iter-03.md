# Mission

Replace the Claude sync/guard/cleanup implementation with the clean per-rule
`SessionStart` hook-output design and validate that the full Loom rule corpus is
visible in same-session Claude context.

# Bound Context

Target ticket: `ticket:cldrel01`.

Operator decision: proceed with the per-rule hook-output route. The user accepts
that `01-core-identity.md` is the only rule that strongly needs to lead; the rest
can be source-marked and need not be perfectly ordered if Claude hook concurrency
reorders them.

Evidence:

- Monolithic full-corpus hook stdout and structured additional context were
  previewed/truncated.
- Plugin-root `CLAUDE.md` and `.claude/rules/loom.md` did not load under local
  `--plugin-dir`.
- Arbitrary 26-chunk structured hook context worked but was too complex.
- A seven-command per-rule raw stdout probe exposed all seven rule files without
  preview/truncation. The observed order was non-sequential because hooks run
  concurrently.

Docs/source support:

- Claude docs say hook output injected into context is capped at 10,000
  characters.
- Local Claude Code source snapshot showed `MAX_HOOK_OUTPUT_LENGTH = 10000`.
- Current rule files are all below 10,000 characters.
- Source inspection showed matching hooks run concurrently and yield as they
  finish, so use staggered sleeps as best-effort ordering, not a strict contract.

# Source Snapshot

Current product files still use the accepted prototype shape:

- `hooks/hooks.json` runs `scripts/claude-sync-rules.sh` on `SessionStart` and
  `scripts/claude-loom-restart-guard.sh` on `UserPromptSubmit`.
- `scripts/claude-sync-rules.sh`, `scripts/claude-loom-restart-guard.sh`, and
  `scripts/claude-clean-rules.sh` exist only for the generated-rule model.
- `INSTALL.md` and `examples/adapters/claude-plugin-install/README.md` still
  document sync, restart, and cleanup behavior.

# Task For This Iteration

Implement the per-rule Claude hook-output design.

Expected implementation shape:

- `hooks/hooks.json` has only a `SessionStart` hook group with matcher
  `startup|clear|compact`.
- That group contains one command hook per top-level rule file.
- Each command emits a source marker and cats exactly one rule file from
  `${CLAUDE_PLUGIN_ROOT}/rules/<filename>`.
- Each command output remains under Claude's documented 10,000-character cap.
- Use small increasing `sleep` delays to make numeric order likely in practice,
  especially ensuring `01-core-identity.md` emits first. Do not claim Claude
  guarantees order.
- Remove `scripts/claude-sync-rules.sh`, `scripts/claude-loom-restart-guard.sh`,
  and `scripts/claude-clean-rules.sh` if no longer referenced.
- Update `INSTALL.md` and `examples/adapters/claude-plugin-install/README.md` to
  describe same-session per-rule hook loading, source markers, no generated rule
  files, no restart guard, and the remaining limitations.
- Update `evidence:claude-sessionstart-stdout-context` with implementation
  validation results.

Keep canonical `rules/`, `skills/`, and `commands/` unchanged.
