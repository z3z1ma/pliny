# Mission

Simplify the Claude plugin prototype by replacing generated rule-file
synchronization with same-session `SessionStart` hook context if the full Loom
rule corpus validates under Claude Code.

# Bound Context

Target ticket: `ticket:cldrel01`.

Governing initiative and plan: `initiative:loom-install-experience` and
`plan:install-experience-harness-adapters` want harness-native package adapters
without turning adapter files into Loom semantic owners.

Prior accepted prototype: `ticket:q7h1d05q` accepted a Claude hybrid plugin that
used `hooks/hooks.json` to run `scripts/claude-sync-rules.sh` on `SessionStart`
and `scripts/claude-loom-restart-guard.sh` on `UserPromptSubmit`. That prototype
generated `loom.md` into `.claude/rules/loom/` or `~/.claude/rules/loom/` and
blocked the first unsafe prompt after bootstrap. It worked locally but carried a
lot of incidental complexity.

New evidence: `evidence:claude-sessionstart-stdout-context` observed Claude Code
2.1.119 seeing raw `SessionStart` stdout in same-session context from a temporary
plugin command that ran `cat "${CLAUDE_PLUGIN_ROOT}"/rules/*.md`.

Inspiration: `obra/superpowers` uses a Claude plugin `SessionStart` hook with
matcher `startup|clear|compact`, `async: false`, and a hook script that emits
structured `hookSpecificOutput.additionalContext`. The superpowers pattern is an
inspiration, not a Loom truth owner.

Git execution context: current checkout on branch `main`, dirty with existing
Codex research and Claude evidence/ticket updates. Do not fetch, edit Git config,
reset, checkout away changes, or commit.

  `"${CLAUDE_PLUGIN_ROOT}/scripts/claude-sync-rules.sh"` and

  describe the old sync/guard/cleanup model.
- `evidence:claude-sessionstart-stdout-context` contains the marker-file probe;
  update it with this iteration's full-rule observations.

Important constraints:

- Preserve canonical top-level `rules/`, `skills/`, and `commands/` unchanged.
- Preserve Claude plugin skills/commands exposure.
- Keep `hooks/hooks.json` as the standard auto-loaded plugin hook file.
- The `cat` command must allow shell glob expansion. Correct pattern:
  `cat "${CLAUDE_PLUGIN_ROOT}"/rules/*.md`; do not quote the `*.md` inside the
  variable expansion.
- Prefer the smallest implementation. Do not add new helper scripts unless the
  validation proves raw `cat` cannot safely do the job.
- If raw stdout has quoting or ordering problems, compare against structured
  `hookSpecificOutput.additionalContext` and implement the smallest working
  alternative.
