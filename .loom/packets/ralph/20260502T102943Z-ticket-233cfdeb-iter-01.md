# Task For This Iteration

Implement the smallest coherent operator-guardrail pass:

1. Add an explicit warning against generic `scratch.md`, `notes.md`, or
   junk-drawer files when the material actually belongs to an owner layer.
2. Route scratchpad-like material to the correct owner: tickets for live state and
   next route, evidence for observations/validation, research for synthesis and
   rejected options, wiki for accepted explanation, memory for support-only recall,
   and specs/plans/constitution when those owners apply.
3. Expand external-reference lifecycle guidance so issue trackers, PRs, URLs,
   dashboards, generated context files, harness artifacts, and package/release
   surfaces can request, mirror, package, or navigate Loom work without owning
   live state, intended behavior, evidence sufficiency, critique verdicts, or
   closure.
4. Add file-first concurrency guidance: re-read before writing, inspect current
   git/worktree state when relevant, preserve unrelated changes, fail closed on
   overlapping edits or stale source, and route stale/contradictory records to the
   owner layer rather than latest-file-wins behavior.
5. Put guidance where future agents will actually read it. Prefer small owner
   surface edits over broad duplication.
