# Page Types

## Concept page

Use for ideas, abstractions, and mental models.

Examples:

- Ralph
- packet discipline
- the outer loop
- ticket truth

## Workflow page

Use for repeatable procedures.

Examples:

- running one Ralph iteration
- reconciling critique into tickets
- promoting accepted understanding into the wiki

## Reference page

Use for compact operational references.

Examples:

- ticket states
- packet fields
- scope conventions
- naming rules

## Codebase atlas page

Use for accepted repository or module structure.

Codebase atlas pages explain structure. They do not own behavior, policy, or
live execution.

Recommended sections:

- Summary
- Major Modules
- Important Entry Points
- Test Surface
- Build / Validation Commands
- Data / Persistence Boundaries
- Risky Areas
- Source Records
- Last Verified

## A simple rule

If the page is mostly about *what something means*, use a concept page.

If the page is mostly about *how to do something*, use a workflow page.

If the page is mostly about *fields, shapes, or quick lookup*, use a reference page.
