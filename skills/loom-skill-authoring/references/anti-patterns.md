# Anti-Patterns

Avoid skills that are:

- too broad to give good procedure
- too narrow to justify their own subsystem
- overlapping heavily with another skill
- vague about what they own
- descriptions that summarize the whole workflow instead of naming activation
  triggers
- pure inspiration with no operational content
- dependent on a hidden runtime the user cannot inspect
- force-loading heavy references when conditional reads would preserve context
- narrative one-offs that should be research, wiki, memory, or a ticket note
- discipline rules with no pressure-test, critique, or evidence posture
