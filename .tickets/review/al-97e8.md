# Core: extract shared exec/fs/time helpers used by workspace

Workspace currently carries small utility wrappers (`workspace/utils.py`, `workspace/time_utils.py`, `workspace/duration.py`, plus fs-safe encoding in `workspace/state.py`). Some of these already delegate to `agent_loom.core.*`, but the layering isn't consistent and other subsystems have their own parallel helpers.

Goal: move truly cross-cutting primitives into `src/agent_loom/core/` and make workspace (and later other subsystems) depend on them.

## Design

Candidate extractions into `agent_loom.core`:
- `core.exec` (or `core.subprocess`): a single `run()` wrapper that captures stdout/stderr, supports `check`, and raises a rich exception type.
- `core.concurrent` (or `core.parallel`): `parallel_map(fn, items, jobs)`.
- `core.fs`: `ensure_dir()`, plus a reversible `fs_escape()` / `fs_unescape()` for path segments.
- `core.time`: extend `parse_iso()` to accept trailing `Z` and add `parse_iso_z` alias; extend `parse_duration_seconds()` to support `w` (weeks) if we want it globally.

Workspace changes:
- Delete or drastically shrink `src/agent_loom/workspace/utils.py`, `src/agent_loom/workspace/time_utils.py`, `src/agent_loom/workspace/duration.py`.
- Replace `workspace.state.fs_escape/fs_unescape` with `core.fs.*`.
- Ensure all callers are updated (workspace + dashboard workspace_read + any other imports).

Non-goals:
- Do not change external CLI behavior.
- Do not change workspace metadata semantics.

Notes:
- `agent_loom.core.git` currently overlaps with workspace's git plumbing; keep that as a separate follow-up ticket (don’t mix concerns here).

## Acceptance Criteria

- Workspace no longer defines its own duration/ISO parsing utilities.
- Shared helpers live in `src/agent_loom/core/` with unit tests.
- Gates pass: `uv run basedpyright`, `uv run ruff check .`, targeted `uv run pytest` for workspace + any new core tests.
