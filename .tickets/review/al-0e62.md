# Workspace: lifecycle/annotations schema v2 + clean metadata storage

Workspace's differentiator is metadata + lifecycle: purpose/ticket/owner/TTL, last-used tracking, and safe cleanup.

Today metadata works, but the storage layout and schema boundaries are messy:
- Repo mode stores worktree metadata JSON files alongside actual worktree directories (`.loom-repo/worktrees/*.json`).
- TTL expiration logic is duplicated.
- `last_used_at` is only updated opportunistically.

This ticket makes lifecycle metadata a first-class, coherent subsystem.

## Design

Storage layout proposal:
- Repo mode internal dir:
  - `.loom-repo/worktrees/<branch>/` (directories only)
  - `.loom-repo/meta/worktrees/<branch>.json` (metadata only)
  - `.loom-repo/states/<snapshot>.json`
- Poly mode stays conceptually similar:
  - `worktrees/<group>/<repo>/` (directories)
  - `.loom/meta/groups/<group>.json` (or `.loom/worktrees/<group>.json`, but pick one and keep it clean)

Schema proposal (common across repo + poly):
- `kind`: normal|sandbox|...
- `purpose`: string
- `ticket_id`: string (optional)
- `owner`: structured object + optional override string
- `ttl_seconds`: int (optional)
- timestamps: created_at, updated_at, last_used_at
- optional lifecycle flags: pinned, archived (avoid overfitting)

Behavior:
- Every command that *uses* a worktree or group should touch `last_used_at` (status/diff/exec/check-clean/etc).
- Centralize TTL expiration computation in a single module (e.g. `workspace/lifecycle.py`).
- Provide a small API:
  - `meta_get_*`, `meta_touch_*`, `meta_set_*`, `meta_list_*`, `meta_prune_*`.

Migration strategy:
- Since Loom is greenfield, we can do a one-time migration on first access:
  - If old paths exist, read+write into new locations.
  - Keep migration logic isolated and tested.

Non-goals:
- Do not add new end-user commands here beyond what's needed to validate the new layout.

## Acceptance Criteria

- Repo mode no longer mixes metadata JSON files with worktree directories.
- TTL expiration and meta touch logic are centralized and reused by cleanup/GC/sandbox.
- All workspace tests pass, plus at least one new test that asserts the new metadata storage locations.
- Gates pass: `uv run basedpyright`, `uv run ruff check .`, `uv run pytest tests/test_workspace_metadata.py tests/test_workspace_leases_gc.py`.
