# Ticket: UX polish + flexible input normalization

Improve `loom ticket` usability by accepting flexible inputs (priority/status/id refs) while keeping canonical stored/output forms.

## Acceptance Criteria

- Priority flags accept 0..4, P0..P4, and high/medium/low (etc) and normalize to int.
- Status inputs accept common aliases and normalize to open|in_progress|closed.
- Ticket refs accept #id, id.md, and .tickets/id.md forms.
- `create` supports `--title` with clear precedence rules.
- Add tests covering the above; run ruff, basedpyright, and pytest.
